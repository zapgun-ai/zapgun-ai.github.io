why

The cache is rigged to evict

“Show me the incentive and I'll show you the outcome.” — Charlie Munger

Anthropic's pricing rewards cache eviction. Cache-creation tokens cost roughly 10× cache-read tokens, and the prompt cache silently evicts at three boundaries the SDK docs never mention. Every eviction makes you pay full freight to rebuild a prefix you already sent. clawback.md captures your system + tools prefix and keeps it warm across each one.

5-minute idle decay

keep-alive

The default ephemeral cache expires after ~5 minutes idle. clawback sends synthetic pings every 1–4 minutes so the prefix stays warm between turns.

Date rollover

strip-ephemeral

Claude Code injects today's date into the system prompt; at midnight the bytes change, the cache key changes, everything rebuilds. clawback normalizes the date and <env> blocks so the hashed bytes don't move across days.

Billing-token rotation

normalize

A per-request token (cch=<hex>) rotates every call and fragments the cache key. clawback normalizes it so consecutive turns land on one shared key instead of a fresh one each time.

the durable point

You pick the cache tier — not your client

“clawback doesn't claim 5-minute caching is always cheaper — it claims the cache tier should be your call, not a silent default your client can change under you.”

Claude Code already writes the prompt cache at the 1-hour tier on its own — undocumented, where the SDK describes a 5-minute default. It's not a clawback artifact; it's what your client sends. That's a write-price change you couldn't see and couldn't opt out of.

clawback pins the tier deterministically at the proxy: force the documented 5m on tight loops where the 1h premium buys nothing, or inject 1h on purpose across real idle gaps where it earns its keep. Either way, the wheel is in your hands.

the controls

The seven knobs

Every knob is one click in the web UI or one POST to an admin endpoint. passthrough is the baseline arm — it turns everything off so you can A/B clawback against doing nothing.

Source of truth: the project README → The Seven Knobs.
Knob What it does Why it might help
passthrough Turns every clawback.md intervention off (baseline arm) Measure clawback.md's effect — quick A/B against doing nothing
keep-alive Synthetic pings to keep the prompt cache warm Avoid 5-minute cache expiry between turns
1h cache TTL Use Anthropic's extended cache tier Make cache survive long idle gaps
strip-ephemeral Normalize dates / env in system prompts Prevent silent cache invalidation across days
mobile gzip outgoing + non-streaming responses Cut radio-on time on tethered/slow links
extended keep-alive 15–45 min cadence (pairs with 1h TTL) Cut keep-alive cost ~6–12×
auto-continue Resume claude after rate-limit cap resets Don't lose your seat at the table

verify

Don't trust us — run it yourself

clawback ships with the rig that measures its own claim. The proxy is zero-install; the benchmark harness lives in the repo, so clone it when you want to A/B your own traffic.

zsh — clawback 
$ npx clawback quickstart
  ▸ wrote ./clawback.md   ▸ patched ~/.claude/settings.json
  ▸ proxy on http://localhost:8080   ▸ launching claude…

$ open http://localhost:8080/_proxy/ui/         # watch tokens live

$ scripts/ab_block.sh --profile L2 --turns 200   # A/B vs doing nothing
  arm A0  passthrough (baseline)
  arm A5  keep-alive + 1h-TTL + strip-ephemeral

$ open http://localhost:8080/_proxy/report       # read your own numbers

savings %: insufficient  until ≥30 turns/arm — never a number it can't defend

There is no hero percentage on this page on purpose. Below 30 turns per arm the analyzer prints insufficient instead of a figure it can't stand behind, and every number it does print carries a 95% confidence interval. No headline savings figure has cleared that significance gate yet — so we don't quote one. Run it against your own sessions and read your own report.
